# گالری نوول — راهنمای بالا آوردن لوکال (روی سیستم خودتان)

این پوشه شامل دو بخش است:
```
nouvelle-gallery-project/
├── backend/    سرور Node.js + Express + MongoDB
└── frontend/   فایل index.html که به بک‌اند وصل می‌شود
```

## مرحله ۱ — نصب MongoDB
یکی از این دو راه را انتخاب کنید:
- **ساده‌ترین راه:** یک حساب رایگان در [MongoDB Atlas](https://www.mongodb.com/atlas) بسازید و آدرس اتصال (connection string) بگیرید.
- یا MongoDB را روی سیستم خودتان نصب کنید: https://www.mongodb.com/try/download/community

## مرحله ۲ — بالا آوردن بک‌اند
```bash
cd backend
npm install
cp .env.example .env
```
فایل `.env` را باز کنید و حداقل این مقادیر را تنظیم کنید:
- `MONGO_URI` → آدرس دیتابیستان (مثلا `mongodb://127.0.0.1:27017/nouvelle-gallery` برای MongoDB محلی)
- `JWT_SECRET` → یک رشته‌ی دلخواه و طولانی
- `ADMIN_USERNAME` و `ADMIN_PASSWORD` → اطلاعات ورود پنل مدیریت

سپس:
```bash
npm run seed     # ساخت کاربر مدیر و ثبت نرخ اولیه طلا (فقط بار اول)
npm run dev      # اجرای سرور روی http://localhost:4000
```
اگر پیام `🚀 سرور گالری نوول روی پورت 4000 در حال اجراست` را دیدید، بک‌اند آماده است.
می‌توانید با باز کردن `http://localhost:4000/api/health` در مرورگر از سالم‌بودن سرور مطمئن شوید (باید `{"ok":true,...}` نمایش داده شود).

## مرحله ۳ — باز کردن فرانت‌اند
فایل `frontend/index.html` را مستقیم با دوبار کلیک در مرورگر باز کنید — نیازی به نصب یا سرور جداگانه نیست.

⚠️ نکته مهم: در ابتدای فایل `frontend/index.html`، خط زیر را پیدا کنید:
```js
const API_BASE = 'http://localhost:4000';
```
اگر بک‌اند را با همین تنظیمات پیش‌فرض بالا آورده‌اید، همین مقدار درست است و نیازی به تغییر نیست.

## مرحله ۴ — تست کامل
1. صفحه اصلی سایت را ببینید — چون هنوز محصولی اضافه نکرده‌اید، بخش «جدیدترین محصولات» خالی است.
2. از دکمه «پنل مدیریت» وارد شوید (با `ADMIN_USERNAME` / `ADMIN_PASSWORD` که در `.env` گذاشتید).
3. از بخش «افزودن محصول»، چند محصول با عکس، وزن و اجرت اضافه کنید.
4. به سایت اصلی برگردید، یک حساب کاربری معمولی بسازید، محصول را به سبد خرید اضافه کنید و سفارش ثبت کنید.
5. در پنل مدیریت → «سفارش‌ها»، سفارش ثبت‌شده را با نام، آدرس و اقلام مشتری ببینید.

## نکته درباره پرداخت
تا وقتی `ZARINPAL_MERCHANT_ID` واقعی را در `.env` بک‌اند نگذاشته‌اید، سفارش‌ها بدون درگاه واقعی پرداخت ثبت می‌شوند (برای تست کامل فرآیند کافی است). جزییات اتصال زرین‌پال در `backend/README.md` توضیح داده شده.

## اگر با خطای اتصال مواجه شدید
- مطمئن شوید ترمینال بک‌اند باز و در حال اجراست (نباید بسته باشد).
- مطمئن شوید `MONGO_URI` در `.env` درست و در دسترس است.
- اگر پورت ۴۰۰۰ توسط برنامه دیگری اشغال است، `PORT` را در `.env` عوض کنید و همان مقدار را در `API_BASE` فرانت‌اند هم به‌روزرسانی کنید.
